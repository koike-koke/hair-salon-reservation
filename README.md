　#美容室向け予約サイト　　
**〜 LINE認証を採用した、美容室予約システム 〜**

　##🌟プロジェクト概要
**「身近な美容師の課題を、技術で解決する」**
フリーランスとして活動する友人の美容師が、SNSのDMで予約を受け、手作業で予約管理アプリに入力しているという課題を知り、その業務を効率化するために開発しました。

　###解決できる課題：
**予約業務の自動化**: DMのやり取りにかかる時間を削減し、24時間自動で予約を受付。
・転記ミスの防止: 顧客自身が入力することで、時間の間違いや転記漏れをゼロに。
・一元管理: LINEログインと連携し、予約から顧客情報の紐付けまでをシステム内で完結。

　技術的なこだわり
１、論理削除の導入
予約のキャンセル処理において、データを物理的に削除せず active フラグで管理する「論理削除」を採用しました。
目的: 「誰がいつキャンセルしたか」という履歴を残すことで、顧客分析や予約が急に消えて困るといった混乱を防ぐためです。

２、予約重複チェックロジック
ユーザーが予約を行う際に、既存の予約と重ならないように実装しました。
判定条件：　(既存予約の開始時刻 < 新規予約の終了時刻) AND (既存予約の終了時刻 > 新規予約の開始時刻)
工夫点: 既にキャンセルされた予約（論理削除済みデータ）は判定対象から自動的に除外するように設計し、正確な空き枠管理を実現しました。

３、LINEログインによる認証
UXの最適化:Spring Security 6 と連携し、LINEログインを実装しました。
マーケティング施策との親和性:独自のパスワード管理を不要にし、離脱率の低いスムーズなログインを実現しました。
将来的に公式LINEアカウントを作成した際に連携を見据え、メールアドレス認証ではなくライン認証を採用しました。

　使用技術
バックエンド: Java 21 / Spring Boot 4.0.1
セキュリティ: Spring Security (OAuth 2.0 / LINE Login)
データベース: MySQL 
ORM: Spring Data JPA
フロントエンド: Thymeleaf / HTML5 / CSS3
ビルドツール: Maven
その他: Lombok

 今後のアップデート予定
1. 柔軟なスケジュール管理機能
定休日や祝日の設定、日ごとの営業時間変更（時短営業など）機能。

3.LINE公式アカウント連携
予約完了通知や前日リマインド、クーポン配布を自動化。
4. UI/UXの刷新（フロントエンドの改善）
スマートフォンからもスムーズに予約操作ができるよう、デバイスに最適化されたレイアウトの導入。
